1.1 Data Description and task classification¶

1.1.1 Introduction¶

Everyone is talking about the amazing wine quality you can find from Portugal. But not everyone knows what exactly they are. Wine certification is generally assessed by physicochemical and sensory tests. It’s really complicated and not easy to understand. What if the quality was given by a straightforward score range from 0 to 10? Or simply by their level such as classical, premium, and luxury? In this project, we dedicate to propose an algorithm to predict the wine quality. Multi-layer perceptron is used to build the model which predict the quality of wine based on the wine's physicochemicales test results.

1.1.2 Data Description¶

The data we used was obtained through UCI Machine Learning Repository. The two datasets in the original link are related to red and white variants of the Portuguese "Vinho Verde" wine. In this project, we only use the white wine dataset to build our model. This dataset contains 12 variables. But not all input variables are relevant to our model. We will perform the variable selection in the late analysis. Also, this dataset contains an important information on wine scoring. Wine samples were evaluated by wine experts. And wine quality scores range from 0 (very bad) to 10 (very excellent). We will set our classes according to these scores. Our data only range form score 3 to 9. So we set score 3 to level 1, score 4 to level 2 and so on. The highest level is level 7 and stands for score 9.

1.1.3 Study Purpose¶

In this project, we developed a model for the wine quality classification using the dataset acquired from UCI Machine Learning Repository. The objective of the model is to benefit both wine certification entities, wine producers or suppliers, and personal buyers in evaluating wine qualities by outputting wine quality levels. The detailed benefits for wine certification entities, wine producer or supplier, and personal buyers are respectively shown as follows:

For wine certification entities: Our model can be used as a supplementary tool for oenologist’s wine evaluation. This model may help them make more accurate and efficient evaluations regarding wine qualities.

For wine producers or suppliers: For the wine producers, our model could give them a clear idea on factors affecting the wine quality and potentially help them improve the wine quality resulting in saving time on production process. For suppliers, this model acts as a criterion for pricing. The supplier can use this model to set their marketing strategy and making profit on the product.

For personal buyers: Most of the personal buyers are not like the oenologist who is really know how to distinguish the wine quality. This model could give them out a straightforward wine qulity levels to help them find a wine that meet their preference and their budgets. Also, the level of wine usually have a positive relationship with the wine price. Personal buyer can use the level of the wine to find a wine under their budget.

2. Evaluation¶

2.1 Evaluation Metric¶

As for evaluation metric, first thing comes into mind is classification accuracy. But it is not always the right choice. Evaluation metric should help us to decide whether our model is good enough to solve our problem. Considering the imbalance between classes in our data, it is obvious that classification accuracy will not give us enough information to make this decision.

Besides classification accuracy, there are three other performance-measuring methods most commonly used: Precision, Recall and F1-score.

Precision is also called Positive Predictive Value, which focus more on classifiers exactness. A low precision indicates a large number of False Positives.

Recall is also called True Positive Rate, which focus more on classifiers completeness. A low recall indicates a large number of False Negatives.

F1 score conveys the balance between the precision and the recall.

In our case, our model aims to predict the quality score of wine. Either upgrading, downgrading or misclassifying the quality of wine will make the prediction result meaningless. So, we want to minimize both FP and FN rate. F1 score matches our goal.

For F1 score, there are still two options for us: micro-averaged f1 score and macro-averaged f1 score.

In Macro-averaging, f1 score is computed locally over each class first and then average over all classes is taken. The result will again be influenced by the imbalance of data. So, micro-averaged F1 score will be chosen, considering the imbalance.

2.2 Data Spliting¶

We use cross validation to split data. The detail process will be shown in following contents.

Before choosing the model, we first consider to compare 2 cross validation method: K-fold and Straitified K-fold. K-fold method equally divided the sample into K groups of small samples. And each small sample will be used as testing data while the other K-1 groups will be used as training data. This means that the whole process will repeated K times. At the same time, all obsercations will be used for both training and testing.

But Straitified K-fold is different. Unlike K-fold equally dividing data, Straitified K-fold consider about imbalance in the sample data set. If one class appears more frequently than others, straitified K-fold will show this imbalance in later spliting.

In our case, it is easily to tell the imbalance between classes. Thus simply using K-fold strategy can be unreasonable. In order to guarantee the imbalance distribution of classes in original sample does cause incorrect validation in result, we use Straitified K-fold in our model.

from scipy.special import expit
import sys
from sklearn.metrics import accuracy_score


class MLPLayerPerceptronBase(object):
    def __init__(self, n_hidden=30,          
                 C=0.0, epochs=500, eta=0.001, random_state=None, layer = 2, phi = 'sigmoid', f_cost = 'quadratic'):
        np.random.seed(random_state)
        self.n_hidden = n_hidden
        self.l2_C = C
        self.epochs = epochs
        self.eta = eta
        self.layer = layer
        self.phi = phi
        self.f_cost = f_cost
        
    @staticmethod
    def _encode_labels(y):
        """Encode labels into one-hot representation"""
        onehot = pd.get_dummies(y).values.T
            
        return onehot

    def _initialize_weights(self):                             
        """Initialize weights with small random numbers."""
        W = [None] * self.layer
        
        W_first_num_elems = (self.n_features_ + 1)*self.n_hidden
        W[0] = np.random.uniform(-1.0, 1.0, size = W_first_num_elems)
        W[0] = W[0].reshape(self.n_hidden, self.n_features_ + 1)
        
        W_last_num_elems = (self.n_hidden + 1)*self.n_output_
        W[self.layer - 1] = np.random.uniform(-1.0, 1.0, size = W_last_num_elems)
        W[self.layer - 1] = W[self.layer - 1].reshape(self.n_output_, self.n_hidden + 1)
        for i in range(1, self.layer - 1):
            W[i] = np.random.uniform(-1.0, 1.0, size=(self.n_hidden + 1)*self.n_hidden)
            W[i] = W[i].reshape(self.n_hidden, self.n_hidden + 1)
            
            
        return W
    
    @staticmethod
    def _sigmoid(z):   
        """Use scipy.special.expit to avoid overflow"""
        return expit(z)      
    def _linear(z):
        return z
    
    @staticmethod
    def _add_bias_unit(X, how='column'):
        """Add bias unit (column or row of 1s) to array at index 0"""
        if how == 'column':
            ones = np.ones((X.shape[0], 1))
            X_new = np.hstack((ones, X))
        elif how == 'row':
            ones = np.ones((1, X.shape[1]))
            X_new = np.vstack((ones, X))
        return X_new
    
    @staticmethod
    def _L2_reg(lambda_, W):           
        """Compute L2-regularization cost"""
        # only compute for non-bias terms
        result = (lambda_/2.0) * np.sqrt(np.mean(W[0][:, 1:] ** 2))
        for i in range(1, len(W)):
            result = result + np.sqrt(np.mean(W[i][:, 1:] ** 2))
        return result
            
        #return (lambda_/2.0) * np.sqrt(np.mean(W1[:, 1:] ** 2) + np.mean(W2[:, 1:] ** 2))
    
    def _cost(self,A_list,Y_enc,W): 
        '''Get the objective function value'''
        if self.f_cost == 'quadratic':
            cost = np.mean((Y_enc-A_list[-1]**2))
        elif self.f_cost == 'cross_entropy':
            cost = -np.mean(np.nan_to_num((Y_enc*np.log((A_list[-1])+(1-Y_enc)+epsilon))*np.log(1-A_list[-1]+epsilon)))
        L2_term = self._L2_reg(self.l2_C, W)       
        return cost + L2_term                   
        #return cost
                                      
    def _feedforward(self, X, W):
        """Compute feedforward step
        """
        A_list = [None] * (len(W) + 1)
        Z_list = [None] * len(W) 
        A = self._add_bias_unit(X, how='column')
        Z = W[0] @ A.T
        
        A_list[0] = A
        Z_list[0] = Z
        
        if self.phi == 'sigmoid':
            for i in range(1, len(W)):
                A_list[i] = self._sigmoid(Z_list[i - 1])
                A_list[i] = self._add_bias_unit(A_list[i], how='row')
                Z_list[i] = W[i] @ A_list[i]

            A_list[len(W)] = self._sigmoid(Z_list[len(W) - 1])                
        
        elif self.phi == 'linear': 
            for i in range(1, len(W)):
                #A_list[i] = self._linear(Z_list[i - 1])
                A_list[i] = Z_list[i - 1]
                A_list[i] = self._add_bias_unit(A_list[i], how='row')
                Z_list[i] = W[i] @ A_list[i]

            #A_list[len(W)] = self._linear(Z_list[len(W) - 1])
            A_list[len(W)] = Z_list[len(W) - 1]
            
        return A_list, Z_list
        
    def _get_gradient(self, A_list, Z_list, Y_enc, W):
        """ Compute gradient step using backpropagation.
        """
        sigma = [None] * len(W)
        if self.f_cost == 'quadratic':
            sigma[-1] = -2 * (Y_enc - A_list[-1]) * (A_list[-1]) * (1 - A_list[-1])
        elif self.f_cost == 'cross_entropy':
            sigma[-1] = (Y_enc - A_list[-1])
        if self.phi == 'sigmoid':
            sigma[-2] = (W[-1].T @ sigma[-1]) * A_list[-2] * (1 - A_list[-2])
            for i in range(len(sigma) - 3, -1, -1):
                sigma[i] = (W[i + 1].T @ sigma[i + 1][1:,:]) * A_list[i + 1] * (1 - A_list[i + 1])
        
        elif self.phi == 'linear':
            sigma[-2] = (W[-1].T @ sigma[-1])
            for i in range(len(sigma) - 3, -1, -1):
                sigma[i] = (W[i + 1].T @ sigma[i + 1][1:,:])

        #sigma[0] = (W[1].T @ sigma[1][1:, :]) * A_list[1] * (1- A_list[1])
        #sigma[1] = (W[1].T @ sigma[2][1:, :]) * A_list[2] * (1- A_list[2])
        #sigma[0] = (W[1].T @ sigma[1][1:, :]) * A_list[1] * (1- A_list[1]) ############################
        grad_list = [None] * (self.layer)        
        grad_list[0] = sigma[0][1:,:] @ A_list[0]
        grad_list[len(grad_list) - 1] = sigma[len(grad_list) - 1] @ A_list[len(grad_list) - 1].T
        for i in range(1, len(grad_list) - 1):
            grad_list[i] = sigma[i][1:, :] @ A_list[i].T
        for i in range(len(grad_list)):            
            grad_list[i][:, 1:] = W[i][:,1:] * self.l2_C
        return grad_list
        
    
    def predict(self, X):
        """Predict class labels"""
        A_list,Z_list = self._feedforward(X, self.W)
        y_pred = np.argmax(A_list[-1], axis=0)
        return y_pred
    
    def fit(self, X, y, print_progress=False):
        """ Learn weights from training data."""
        
        X_data, y_data = X.copy(), y.copy()
        Y_enc = self._encode_labels(y)
       
        self.W = []
        self.n_features_ = X_data.shape[1]
        self.n_output_ = Y_enc.shape[0]
        self.W = self._initialize_weights()

        self.cost_ = []
        self.score_ = []
        for i in range(self.epochs):

            if print_progress>0 and (i+1)%print_progress==0:
                sys.stderr.write('\rEpoch: %d/%d' % (i+1, self.epochs))
                sys.stderr.flush()

          
            A_list,Z_list = self._feedforward(X_data,self.W)
            
            cost = self._cost(A_list,Y_enc,self.W)
            self.cost_.append(cost)

            # compute the gradient
            grad_list = self._get_gradient(A_list, Z_list, Y_enc, self.W)

            self.W[0] -= self.eta * grad_list[0]
            self.W[len(self.W) - 1] -= self.eta * grad_list[len(self.W) - 1]
            for i in range(len(self.W)):
                self.W[i] -= self.eta * grad_list[i]
            self.score_.append(f1_score(y_data,self.predict(X_data),average='micro'))   
        return self
    def get_params(self, deep=True):
        return {
            'eta': self.eta, 'C': self.l2_C,"phi": self.phi,
            'f_cost': self.f_cost, 'layer': self.layer,
            'n_hidden': self.n_hidden, 'epochs': self.epochs
        }
    def set_params(self, **parameters):
        for parameter, value in parameters.items():
            self.__setattr__(parameter, value)
        return self
